KUTX Deployment Guide (Hinglish)
Ye guide aapko help karegi aapke chat platform ko Vercel par live karne mein.

Step 1: GitHub pe Code Push Karna
Mainne aapke liye code GitHub repository (https://github.com/keerti0singh-cmyn/kutx.git) par push kar diya hai.

NOTE

Agar push successful nahi hua hai (authentication issue ki wajah se), toh aapko niche diye gaye commands apne local terminal mein chalane honge:

bash
git remote add origin https://github.com/keerti0singh-cmyn/kutx.git
git branch -M main
git push -u origin main
Step 2: Vercel Project Setup
Vercel par login karein.
"Add New" -> "Project" par click karein.
Apna GitHub account connect karein aur kutx repository ko Import karein.
Step 3: Environment Variables (IMP!)
Vercel deployment ke waqt Environment Variables section mein ye values add karein:

Variable Name	Value (Aapke project se)
NEXT_PUBLIC_SUPABASE_URL	https://oxtomavjxpzjvqbybbfw.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY	eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... (Poora key copy karein)
TIP

Aap ye values apne local 
.env.local
 file se lekar copy-paste kar sakte hain.

Step 4: Build Errors ko Solve Karna
Aapke project mein kuch TypeScript errors hain (
tsc_errors.txt
 mein dekhein). Inko fix karna best hai, lekin agar aap jaldi mein hain aur build bypass karna chahte hain, toh 
next.config.ts
 mein ye add kar sakte hain:

typescript
const nextConfig = {
  typescript: {
    ignoreBuildErrors: true,
  },
  eslint: {
    ignoreDuringBuilds: true,
  },
  // ... existing config
}
Step 5: Deploy!
Saare settings ke baad "Deploy" button par click karein. Vercel automatically build process start kar dega.

Step 6: Supabase Auth Configuration
Site live hone ke baad, aapko Supabase Dashboard mein jaakar Authentication -> URL Configuration mein apni Vercel URL (e.g., https://kutx.vercel.app) ko "Site URL" mein update karna hoga.

Badhai ho! Aapki site ab live honi chahiye. Koi issue aaye toh batayein.
